Voorbeelden van mijn werk

Rijksmuseum toegankelijke ticketflow

Live link GitHub

Voor de meesterproef van mijn minor koos ik de case "Maak de ticketflow van het rijksmuseum toegankelijk voor zoveel mogelijk doelgroepen." omdat dit een uitdaging was op zowel conceptueel als technisch niveau. Uiteindelijk heb ik met een mede CMD'er een mooi product op kunnen zetten waarbij de hele ticketflow navigeerbaar is via het toetsenbord en de datepicker zeer uniek, maar goed werkt. De datepicker neemt een flexibele input als format, denk hierbij aan "vandaag, morgen, volgende week vrijdag, etc.". Dit help mensen met een screenreader enorm, omdat deze niet door irritante calendars heen hoeven te navigeren.

Enkele highlights van dit project

  • Toetsenbord navigeerbaar
  • Screenreader getest, aria labels included
  • Accessible "magic" datepicker
  • Progressive disclosure
  • Entry points op detailpagina's die smart defaults invullen
  • Wekelijkse check-ins met stakeholders en opdrachtgever, eindpresentatie voor opdrachtgever en mede geïnteresseerden
  • Volledige documentatie (debrief, design rationale, iteraties en reflectie)

Browser technologies

Live link GitHub

Hier heb ik een applicatie gemaakt die in 3 lagen werkt: een functional laag, een useable laag en een pleasureable laag. Dit betekent dat de applicatie werkt in vanilla HTML, mooier wordt gemaakt met CSS en pleasureable interacties worden toegevoegd met JavaScript. Het doel van deze oefening was om te leren hoe (en vooral waarom) je functionaliteit behoudt of weg laat vallen wanneer bepaalde technologieën wegvallen.

Progressive Web Application

Live link GitHub

In dit project heb ik een progressive web app gemaakt waarbij men met behulp van een API informatie over voertuigen kan opvragen door een VIN in te voeren. Dit houdt in dat de applicatie zonder JavaScript volledige functionaliteit behoudt. Ook is er een service worker geïmplementeerd die statische bestanden cached en kan serveren. Daarbij is in dit project de critical render path geoptimaliseerd door de css files samen te voegen en te minimaliseren.
